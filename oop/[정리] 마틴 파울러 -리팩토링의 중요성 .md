### Workflows of Refactoring

![1*dBaTqt07F5BT7V2CDK00Zg](https://miro.medium.com/max/1400/1*dBaTqt07F5BT7V2CDK00Zg.png)





+ Add Function
  + 기능 수정, 추가, 삭제 
+ Refactoring
  + 테스트 코드를 실행하며 기능이 이전과 동일하게 동작하면서 개발 진행 
  + 소프트웨어 내부 설계를 더 높은 수준으로 진화시키는 것 
  + 버그를 찾아도 고치면 안됨. 기능이 달라지기 때문
  + 종류
    - litter-picking (쓰레기 줍기)    
    - Comprehension (이해하기 쉽게)     
    - Preparatory (새로운 기능 추가하기 위해 기존의 구조 개선)     
    - Planned (개발 프로세스로 주기적으로 계획된 리팩토링 - 옵션)
